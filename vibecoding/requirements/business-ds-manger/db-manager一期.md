# 功能需求：数据库表结构同步与 CDC 感知

## 1. 背景与目标
在现有的左侧目录结构基础上，增加“业务数据库表配置”模块。旨在实现开发人员自助管理表结构变更，通过审批流自动同步至生产环境，并自动通知大数据平台进行 CDC 配置。

## 2. 用户故事
-   **作为** 后端开发人员
-   **我希望** 在界面上编辑表结构（字段、索引等），并点击“同步到生产”
-   **以便于** 不需要人工联系 DBA 即可自动完成生产环境的表变更，且大数据平台能自动感知。
-   新增一个模块ds的领域在四层结构中编写对应的接口

## 3. 核心功能流程
1.  **表结构编辑**：
    -   支持在测试环境定义/修改表结构。
    -   支持生成 DDL 预览（Diff 对比）。
2.  **发布与审批** (不做)：
    -   点击“同步到生产”按钮。
    -   触发审批工作流（如：接入钉钉/飞书审批或系统内审批）。
3.  **自动执行** (不做)：
    -   审批通过后，系统自动在生产数据库执行 DDL。
4.  **CDC 联动** (不做)：
    -   若该表被标记为“CDC 启用”或“大数据使用”，DDL 执行成功后，自动调用大数据平台 API 注册该表。()

## 4. 数据结构设计建议
需要在现有系统中增加以下配置实体：
-   `ds_config`: 数据源的配置信息，url, username, password

## 5. 接口需求
-   `POST /api/v1/ds`: 添加数据源配置
-   `GET /api/v1/ds`: 获得数据源配置列表
-   `GET /api/v1/ds/{ds}`: 获得数据源配置
-   `PUT /api/v1/ds/{ds}`: 修改数据源配置
-   `GET /api/v1/ds/dbs`: 获取数据源下的数据库列表
-   `POST /api/v1/ds/dbs`: 新增一个数据库
-   `GET /api/v1/ds/dbs/{db}/tables`: 获取数据源下的数据库下的表列表
-   `GET /api/v1/ds/dbs/{db}/tables/{tbl}`: 获取表详情
-   `PUT /api/v1/ds/dbs/{db}/tables/{tbl}`: 修改一个表结构,默认要存在created_at字段updated_at字段delete_at字段,支持索引的修改
-   `POST /api/v1/ds/dbs/{db}/tables`: 新增一个数据源下的表结构,默认要存在created_at字段updated_at字段delete_at字段,支持索引的添加
-   `DELETE /api/v1/ds/dbs/{db}/tables{tbl}`: 删除一个表结构
